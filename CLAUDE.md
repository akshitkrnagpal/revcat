# revcat

The RevenueCat CLI. Go + cobra.

## Working with this repo

- `go run ./cmd/revcat <args>` runs the CLI from source
- `go build -o revcat ./cmd/revcat` produces a single static binary
- `go test ./...` runs tests
- `go vet ./...` lints

## Architecture

- `cmd/revcat` - main() entrypoint, calls commands.Execute()
- `commands/` - cobra commands organized by resource (auth, init, apps, ...)
- `internal/api` - hand-rolled RC v2 REST client (no SDK exists) + OAuth flow
- `internal/auth` - global file / project-local credential stores + resolver
- `internal/project` - revcat.toml loader (committed half of project context)
- `internal/cliutil` - shared command helpers (Client, ResolveProjectID)
- `internal/output` - TTY-aware renderer (table on TTY, JSON when piped)

## Auth

OAuth-only since v0.4. Three credential storage tiers (resolution order):

1. `REVCAT_REFRESH_TOKEN` env (synthesizes a virtual profile, in-memory only)
2. Walked-up `./.revcat/config.json` (written by `revcat init`, gitignored, mode 0600)
3. `~/.revcat/config.json` (written by `revcat auth login`, mode 0600). The encrypted-keyring backend was dropped in v0.6 because the CGO_ENABLED=0 binary couldn't reach the real OS keychain anyway.

Within tier 3, the active profile name is `--profile` flag > `REVCAT_PROFILE` env > `~/.revcat/active` > `default`.

Project id resolution: `--project-id` flag > `REVCAT_PROJECT_ID` env > resolved credential's bound project (local config or env hatch) > walked-up `revcat.toml`.

## Source of truth

Tier 1 (canonical): cobra commands in `commands/` and the api client in `internal/api/`. Plus RC's v2 API docs for what upstream exposes.

Tier 2 (auto-derived from tier 1):
- `revcat <cmd> --help`
- `docs/src/content/docs/reference/cli.md` â€” regenerated by `make docs-cli`
- The auto-header (frontmatter + intro + subcommands table) of every `docs/src/content/docs/commands/<group>.md` â€” regenerated by `make docs-groups`. Each per-group page has an `<!-- AUTOGEN_END -->` marker; everything below is hand-written and preserved across regens.

Tier 3 (hand-maintained): `README.md`, `skills/*/SKILL.md`, `CHANGELOG.md`, `demo/demo.tape`, getting-started/guides pages, and the post-marker prose in per-group command pages.

After any command-tree change:

```sh
make docs            # regenerates tier-2 derived files
make drift-check     # asserts docs/skills don't reference missing commands
```

`make drift-check` is the local-only test that walks the cobra tree and scans every `.md` for `revcat <cmd>` mentions inside code spans, asserting each command path is real. Catches the "doc names a removed subcommand" class. Add allowlisted hypotheticals (`<group>`, `<command>`, etc.) to the map in `commands/drift_test.go`.

When tier 3 conflicts with tier 1, tier 1 wins. To verify "does X command exist," run `revcat <cmd> --help` or read the cobra command, not the docs.
